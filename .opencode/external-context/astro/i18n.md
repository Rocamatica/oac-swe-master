# Internacionalización (i18n) en Astro v5.x

## 1. Soporte oficial de i18n

**Sí, Astro tiene soporte oficial de i18n** desde la versión 4.x+.

El sistema de i18n se configura en `astro.config.mjs` mediante la opción `i18n` y ofrece un módulo `astro:i18n` con utilidades.

**Fuente:** https://docs.astro.build/en/guides/internationalization

### Configuración básica:
```javascript
import { defineConfig } from "astro/config"

export default defineConfig({
  i18n: {
    locales: ["es", "en", "fr"],
    defaultLocale: "en",
    routing: {
      prefixDefaultLocale: false
    }
  }
})
```

## 2. URLs multilingüe

Astro soporta **múltiples estrategias de enrutamiento** para URLs multilingüe:

### Estrategias de routing (`routing.strategy`):
| Estrategia | Descripción | Ejemplo |
|-----------|-------------|---------|
| `"pathname-prefix-always"` | Siempre con prefijo de idioma | `example.com/en/about` |
| `"pathname-prefix-other-locales"` | Prefijo solo para idiomas no default | `example.com/about` (EN), `example.com/es/about` (ES) |
| `"pathname-prefix-always-no-redirect"` | Siempre con prefijo, sin redirects | `example.com/en/about` |
| `"domains-prefix-always"` | Diferente dominio por idioma | `example.com` (EN), `example.es` (ES) |
| `"manual"` | Control manual completo | Gestionado por el desarrollador |

### Ejemplo con configuración de routing:
```javascript
i18n: {
  locales: ["es", "en", "fr"],
  defaultLocale: "en",
  routing: {
    prefixDefaultLocale: true  // /en/about, /es/about, /fr/about
  }
}
```

**Fuente:** https://docs.astro.build/en/guides/internationalization
**Fuente:** https://docs.astro.build/en/reference/integrations-reference

### Dominios por idioma:
```javascript
i18n: {
  locales: ["es", "en", "fr", "ja"],
  defaultLocale: "en",
  domains: {
    fr: "https://fr.example.com",
    es: "https://example.es"
  }
}
```

**Nota:** La estrategia de dominios requiere modo `server` (SSR) con un adapter compatible.

## 3. Traducción de contenido

Astro **no tiene un sistema integrado de traducción**. Los patrones recomendados son:

### a) Content Collections con subdirectorios por idioma:
```
src/content/
  blog/
    en/
      post1.md
      post2.md
    es/
      post1.md
      post2.md
```

Y filtrar por subdirectorio:
```astro
const englishPosts = await getCollection('blog', ({ id }) => id.startsWith('en/'));
const spanishPosts = await getCollection('blog', ({ id }) => id.startsWith('es/'));
```

**Fuente:** https://docs.astro.build/en/recipes/i18n

### b) Archivos de traducción con funciones helper:

```typescript
// src/i18n/ui.ts
export const languages = {
  en: 'English',
  es: 'Español',
};

export const defaultLang = 'en';

export const ui = {
  en: {
    'nav.home': 'Home',
    'nav.blog': 'Blog',
    'nav.about': 'About',
  },
  es: {
    'nav.home': 'Inicio',
    'nav.blog': 'Blog',
    'nav.about': 'Acerca',
  },
} as const;
```

```typescript
// src/i18n/utils.ts
import { ui, defaultLang } from './ui';

export function getLangFromUrl(url: URL) {
  const [, lang] = url.pathname.split('/');
  if (lang in ui) return lang as keyof typeof ui;
  return defaultLang;
}

export function useTranslations(lang: keyof typeof ui) {
  return function t(key: keyof typeof ui[typeof defaultLang]) {
    return ui[lang][key] || ui[defaultLang][key];
  }
}
```

**Fuente:** https://docs.astro.build/en/recipes/i18n

## 4. Funciones del módulo `astro:i18n`

```javascript
import {
  getRelativeLocaleUrl,        // URL relativa para un locale específico
  getAbsoluteLocaleUrl,        // URL absoluta para un locale específico
  getRelativeLocaleUrlList,    // Lista de URLs relativas para todos los locales
  getAbsoluteLocaleUrlList,    // Lista de URLs absolutas para todos los locales
  getPathByLocale,             // Obtener path para un locale
  getLocaleByPath,             // Obtener locale desde un path
  redirectToDefaultLocale,     // Redirigir al locale por defecto
  redirectToFallback,          // Redirigir al fallback
  notFound,                    // Retornar 404 para i18n
  middleware,                  // Middleware de i18n
  requestHasLocale,            // Verificar si la request tiene locale
  normalizeTheLocale,          // Normalizar código de locale
  pathHasLocale,               // Verificar si el path contiene un locale
  toCodes,                     // Convertir a códigos
  toPaths                      // Convertir a paths
} from 'astro:i18n';
```

### Ejemplo de uso de getRelativeLocaleUrl:
```astro
---
import { getRelativeLocaleUrl } from 'astro:i18n';
const aboutURL = getRelativeLocaleUrl("es", "about");
---
<a href={aboutURL}>Acerca</a>
```

## 5. Mapas de rutas traducidas

Se pueden crear mapas de rutas para tener URLs diferentes en cada idioma:

```typescript
export const routes = {
  en: {
    'about': 'about-us',
    'contact': 'get-in-touch',
  },
  es: {
    'about': 'nosotros',
    'contact': 'contacto',
  },
};
```

Y usar el helper `useTranslatedPath`:
```typescript
export function useTranslatedPath(lang: keyof typeof ui) {
  return function translatePath(path: string, l: string = lang) {
    const pathName = path.replaceAll('/', '');
    const hasTranslation = defaultLang !== l &&
      routes[l] !== undefined && routes[l][pathName] !== undefined;
    const translatedPath = hasTranslation ? '/' + routes[l][pathName] : path;
    return !showDefaultLang && l === defaultLang
      ? translatedPath
      : `/${l}${translatedPath}`;
  }
}
```

**Fuente:** https://docs.astro.build/en/recipes/i18n

## 6. Integraciones para i18n

**NO SE ENCONTRARON** integraciones oficiales de Astro específicas para i18n más allá del módulo `astro:i18n` y las recetas documentadas. Astro proporciona las herramientas base (módulo `astro:i18n`, configuración de routing) pero no existe un paquete tipo `@astrojs/i18n` que automatice traducciones o gestión de contenido multilingüe.

El enfoque recomendado es usar las utilidades de `astro:i18n` combinadas con Content Collections y archivos de traducción manuales.
